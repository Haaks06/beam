# Beam for iOS (Shortcuts setup)

iOS doesn't let a website register as a native Share Sheet target the way
Android PWAs can, so on iOS, **Beam** works through Apple Shortcuts —
appearing in the Share Sheet like any other app when you share a link or a
photo.

## Why there's no one-tap "Add to iPhone" button

A hand-built `.shortcut` file downloaded directly from a website (as
opposed to a link generated by Apple's own iCloud sharing pipeline) hits a
hard wall on current iOS: **"Importing of unsigned shortcuts is not
supported."** This isn't a bug in the file's structure — it's Apple
requiring a real cryptographic signature that only gets attached when a
shortcut is shared via **Share → Copy iCloud Link** from inside the actual
Shortcuts app on a real device. There's no Mac or iPhone anywhere in this
project's toolchain to produce that signature, so a raw downloadable file
is a dead end regardless of how it's built — this was tried and confirmed
broken, not just theorized.

**The real fix**: build one of the shortcuts below on an actual iPhone
(~5 minutes, just the Shortcuts app's own editor — guaranteed to work),
then use **Share → Copy iCloud Link** on it. That link *is* signed and
*is* one-tap-importable for anyone. Hand that link over and the site's
"Get Beam" section can link straight to it — turning this into a real
one-tap install for every future visitor, at the cost of one person doing
the setup once.

## How this avoids the 2-minute session limit being painful

Beam's pairings are ephemeral: a phone and PC pair, get a 2-minute window to
send/receive, then the relay forgets everything until they pair again. A
token fetched *ahead of time* and pasted into a shortcut would go stale the
moment that window closes — racing a 2-minute clock by hand is exactly the
kind of thing worth avoiding.

Instead, both shortcuts claim the pairing code **and** send the item in the
same run, back-to-back with no user pause in between — from tapping Share to
the item landing on your PC is a couple of network round-trips, nowhere near
2 minutes. The only thing with a real shelf life is the *pairing code*
itself, which is good for 10 minutes before anyone claims it — plenty of
time to open Beam on your PC, glance at the code, then go share something.

## Manual build (fallback, or if you'd rather build it yourself)

Two separate shortcuts — one for links, one for photos — each a straight
line of steps with no branching, so there's less to get wrong building it
by hand than the old single-shortcut-with-an-If-branch design.

### "Beam Link to PC"

1. Shortcuts app → **+** → name it `Beam Link to PC`.
2. Tap the shortcut's settings (ⓘ) → **Show in Share Sheet** → on. Under
   "Share Sheet Types", enable **URLs**/**Safari web pages** (and **Text**
   if you also want to share plain notes).
3. Add action **Ask for Input** → Input Type: Text → Prompt: "Beam pairing
   code (from your PC's Beam window)".
4. Add action **Set Variable** → name it `PairingCode`.
5. Add action **Get Contents of URL**:
   - URL: `https://beamlot.com/pair/claim?pairingCode=` then insert the
     `PairingCode` variable, then type `&label=iPhone`
   - Method: POST
   - (no headers, no request body needed)
6. Add action **Get Dictionary Value** → Key: `token`.
7. Add action **Set Variable** → name it `Token`.
8. Add action **Get Contents of URL**:
   - URL: `https://beamlot.com/items/link?token=` then insert the `Token`
     variable, then type `&url=` then insert the `Shortcut Input` variable
   - Method: POST
9. Optionally add **Show Notification** → "Beamed!" at the end.
10. Save. On your PC, open Beam and note the pairing code it's showing
    (click **Start over** first if it's mid-session with something else) —
    then on your iPhone, share a link from any app → Share → "Beam Link to
    PC" → enter that code when prompted.

### "Beam Photo to PC"

Same shape, swap step 8 for the photo endpoint and a file body instead of
a URL parameter:

1. Shortcuts app → **+** → name it `Beam Photo to PC`.
2. **Show in Share Sheet** → on, enable **Images** only.
3. Add action **Ask for Input** → Text → Prompt: "Beam pairing code (from
   your PC's Beam window)".
4. **Set Variable** → `PairingCode`.
5. **Get Contents of URL** → same claim URL/method as above (step 5 in the
   Link shortcut).
6. **Get Dictionary Value** → Key: `token`.
7. **Set Variable** → `Token`.
8. **Get Contents of URL**:
   - URL: `https://beamlot.com/items/photo?token=` then insert the `Token`
     variable
   - Method: POST
   - Request Body: **File**, value = `Shortcut Input`
9. Optionally add **Show Notification** → "Beamed!".
10. Save and use the same way — Share a photo → "Beam Photo to PC" → enter
    the code.

## Notes

- Nothing sensitive is hardcoded into either shortcut (the pairing code you
  type each time is single-use and dead within 10 minutes even if unused) —
  safe to export/share the shortcut itself (e.g. via "Copy iCloud Link")
  without scrubbing anything first.
- A pairing code is single-use: if you mistype it or reuse an
  already-claimed one, the claim response won't have a `token` key and the
  next step fails visibly rather than silently sending nowhere. For a
  friendlier failure message, add an **If** checking whether `Token` has
  any value, and **Show Alert** with the claim response's `error` field
  when it doesn't.
- Test against an `ngrok` URL before pointing a hand-built shortcut at your
  real deployed relay, per the main README's verification steps.
- `relay-server/routes/pair.js` and `relay-server/routes/items.js` accept
  these values as URL query parameters specifically so building these
  shortcuts never requires a JSON or multipart request body — both far
  more fiddly to get right by hand in the Shortcuts editor than a plain
  URL with an inserted variable.
